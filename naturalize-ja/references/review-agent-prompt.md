# Subagent Review Prompt Template

`naturalize-ja` の Step 3 でサブエージェントを起動する際のプロンプトテンプレート。Step 3 では Claude が呼び出し側として、ここのテンプレートに `{...}` を埋めた完成プロンプトを Agent ツールに渡す。

## 重要なポイント

サブエージェントは **独立コンテキスト** で動くため:

- ファイルパスを「ここを読んで」と言っても、確実に拾える保証がない
- このため、`ai-japanese-patterns.md` の **全文** と、監査対象テキストを、プロンプトの中に直接埋め込む
- 監査対象が大きい場合はセクション単位に分割して複数 Agent を並列起動する (各 Agent のコンテキストに無理なく入る分量を 1 ジョブにする)

## プロンプトテンプレート

呼び出し側はこのテンプレートを使い、`{PATTERNS_FILE_CONTENT}` には `references/ai-japanese-patterns.md` の全文を、`{TARGET_TEXT}` には監査対象のテキストを差し込む。`{TARGET_LABEL}` はファイル名や「貼り付けテキスト」など、findings 出力で参照する識別子。

```
あなたは「AI 生成日本語の検出」専門のレビュアです。
これから渡す日本語テキストを、ネイティブ日本語話者が読んで
「これは AI 生成だ」と気づくかどうかの観点で監査してください。

監査基準は以下の禁止フレーズ辞書 (A-O カテゴリ) です。
事前にこれを全部読んでから監査に入ってください。

---
{PATTERNS_FILE_CONTENT}
---

監査対象テキスト ({TARGET_LABEL}):

---
{TARGET_TEXT}
---

出力フォーマットは厳密に次の形を守ってください。

[CRITICAL]
カテゴリ: <カテゴリ記号 (A-O)> - <カテゴリ名>
場所: <{TARGET_LABEL}:行番号 または 段落 N>
元: 「<該当フレーズ>」
案: 「<置換後フレーズ>」
理由: <どの AI シグナルか、辞書のどの行に該当するか>

[IMPORTANT]
...

[NICE-TO-HAVE]
...

ルール:

1. 深刻度の判定基準
   - CRITICAL: ネイティブが読んで「AI 生成だ」と直感する決定打となる表現
   - IMPORTANT: 文章のクオリティに明確に効く改善
   - NICE-TO-HAVE: 好みや微調整レベル

2. 辞書 (A-O) のいずれにも該当しないが「明らかに AI 臭がする」と感じる
   表現があれば、[NEW-PATTERN] セクションを設けて以下の形式で報告:

   [NEW-PATTERN]
   場所: <{TARGET_LABEL}:行番号>
   元: 「<該当フレーズ>」
   なぜ AI っぽいと感じるか: <観察した特徴>
   どのカテゴリに近いか: <A-O のいずれか or 「新カテゴリ提案」>

3. 並べる順序は深刻度順 (CRITICAL → IMPORTANT → NICE → NEW-PATTERN)、
   同深刻度内は場所順 (行番号昇順)

4. 最後に必ずサマリ行を含める:

   SUMMARY:
   CRITICAL: <件数>
   IMPORTANT: <件数>
   NICE-TO-HAVE: <件数>
   NEW-PATTERN: <件数>
   TOTAL: <件数>

5. ヒット 0 のカテゴリでも、深刻度の見出し ([CRITICAL] など) は省略可。
   全体で 0 件なら「監査結果: AI 臭は検出されませんでした」とだけ返す
```

## 並列実行のときの分割方針

監査対象が大きい (e.g. 1 ファイルで 3000 字超え) 場合、Claude は呼び出し側で複数 Agent に分割する:

- セクション (h2 単位) で分割するのが標準
- 各サブエージェントには「自分が監査しているのは『セクション X 〜 Y』である」と明示
- 全 Agent 完了後、Claude が findings を統合して重複削除

並列実行例 (Claude が 1 メッセージで投げる Agent 呼び出し):

```
Agent(subagent_type=general-purpose, prompt=<セクション 1-3 用プロンプト>)
Agent(subagent_type=general-purpose, prompt=<セクション 4-6 用プロンプト>)
Agent(subagent_type=general-purpose, prompt=<セクション 7-9 用プロンプト>)
```

3 つは独立なので並列で投げる。各々の出力を統合して Step 4 (分類) へ。

## サブエージェント結果の取り扱い

- サブエージェントが提案した「案」(置換後フレーズ) は、必ず Claude が文脈に照らして妥当性を再確認する
- 文意を変えてしまう案、不自然になっている案は採用しない
- 「採用見送り」を選んだ場合は、ユーザーへの報告に含める
