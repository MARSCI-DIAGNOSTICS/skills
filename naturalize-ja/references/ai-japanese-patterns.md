# AI-Japanese Patterns: 禁止フレーズ辞書

ここに並ぶフレーズと文型は、生成 AI の出力に頻出する「AI 臭」のシグナル。原則として **すべて避ける**、または置換する。

## 由来

- スキル運用での実フィードバック (ユーザーが直接「これ AI っぽい」と指摘したケースを継続的に蓄積)
- 公開されている AI 文体研究:
  - [【保存版】AIっぽい文章表現大全｜もとやま](https://note.com/yusuke_motoyama/n/n2a1218636f56)
  - [なぜ生成AIが書いた文章に違和感が生まれるのか?｜malna](https://malna.co.jp/blog/why_genai_writing_feels_off/)
  - [ChatGPTが書いた文章を見抜く方法 | AI総合研究所](https://www.ai-souken.com/article/detecting-chatgpt-generated-text)
- textlint-rule-preset-ai-writing の検出パターン

## カテゴリ別の禁止フレーズと置換

### A. メタ説明・冗長な前置き

| 禁止 | 置換 |
|---|---|
| 「まずスタート地点をそろえます」 | 削除して定義から入る |
| 「結論から言うと、〜です」 | 結論を断定で書く |
| 「以下の N 個の観点から説明します。それぞれ順に見ていきましょう」 | 見出しだけで構造を示す |
| 「ここから〜していきます」「これから〜を見ていきます」 | 削除 |
| 「Step 1:」「STEP」「Step」のような英語整序 | 「① 〜」など日本語の通し番号 |
| 「お疲れさまでした」「お読みいただきありがとうございます」 | 削除 |

### B. 擬人化 (anthropomorphic verbs)

| 禁止 | 置換 |
|---|---|
| 「答えてくれる」「教えてくれる」「出してくれる」 | 「出力される」「返す」「示す」 |
| 「〜が登場します」「〜が活躍します」 | 「〜を使います」「〜が現れます」 |
| 「モデルが頑張って」「AI が判断してくれる」 | 「モデルが〜する」「AI が判定する」 |
| 「振り分けてもらう」「教えてもらう」 | 「振り分けさせる」「分類する」 |
| 「導いてくれる」 | 「導く」「示す」 |

### C. 同意の押し付け

| 禁止 | 置換 |
|---|---|
| 「〜ですよね」「〜ですね」 | 削除 or 「のはずです」 |
| 「いかがでしたか?」 | 削除 |
| 「ぜひ〜してみてください」 | 「〜してください」「〜することを推奨します」 |
| 「みなさん」「みなさま」 | 「読者」「利用者」 or 削除 |
| 「お分かりいただけたでしょうか」 | 削除 |

### D. フィラー (内容を表していない単語)

| 禁止 | 置換 |
|---|---|
| 「ポイント」「キモ」「ミソ」「主役」「目玉」 | 該当の内容を直接書く |
| 「気持ち」 (例:「式の気持ちを読み解く」) | 「意味」「役割」 |
| 「イメージです」「〜なイメージ」 | 削除して具体に置換 |
| 「ピンとくる」「腑に落ちる」 | 「理解できる」「把握できる」 |
| 「コツ」「裏ワザ」 | 「方法」「手順」 |
| 「ちょっとした」「ほんの少し」 | 必要なら数値で示す、不要なら削除 |
| 「いわゆる」 | 削除 (本当に必要な場合だけ残す) |

### E. 過剰な強調

| 禁止 | 置換 |
|---|---|
| 「驚くほどシンプル」「とんでもなく速い」 | 「シンプル」「速い」(形容詞のみ) |
| 「とても」「非常に」「めちゃくちゃ」「やたら」 | 削除 |
| 「!」(感嘆符) | 通常は不要 |
| 「本当に」「ものすごく」 | 削除 |

### F. 業務文書 / ビジネス臭

| 禁止 | 置換 |
|---|---|
| 「〜を実現します」「〜を達成します」 | 「〜できます」「〜します」 |
| 「〜に展開できます」「N 個の用途に展開できます」 | 「〜に使えます」「N 個に使えます」 |
| 「〜と呼ばれるものです」 | 「〜です」 |
| 「〜にあたります」「〜に該当します」 | 「〜です」 |
| 「〜ということが言えます」「〜と考えられます」 | 「〜です」「〜になります」 |
| 「〜を実施する」 | 「〜する」 |
| 「ご質問ありがとうございます」 | 削除 |
| 「最適化を行う」「対応を行う」 | 「最適化する」「対応する」 |

### G. 接続詞過多

| 禁止 | 置換 |
|---|---|
| 「まず」「つまり」「そして」「また」「さらに」を文頭で連発 | 削除可能なら削除。文意で論理を保つ |
| 「これにより」 | 「これで」「結果として」 |
| 「したがって」「そのため」「結果として」連鎖 | 1 文に 1 つに絞る |
| 「次に」「続いて」連発 | 1 セクション内で 1 回まで |

### H. 責任回避・曖昧化

| 禁止 | 置換 |
|---|---|
| 「〜と考えられます」「〜することが望ましいでしょう」 | 「〜です」「〜してください」(立場を明示) |
| 「一概には言えませんが、〜」 | 削除 or 条件次第なら条件を具体化 |
| 「メリットもあればデメリットもあります」 | 各々を具体的に列挙 |
| 「(ただし状況によります)」 | 削除 |
| 「〜と言えるでしょう」 | 「〜です」 |
| 「〜の可能性があります」(条件付きでは可) | 確度を明示するか削除 |

### I. ステレオタイプな比喩

| 禁止 | 置換 |
|---|---|
| 「レシピ」「土台」「DNA」「車の両輪」「羅針盤」「エンジン」「スパイス」「潤滑油」 | 比喩を最小限にする or 業界の具体例に置換 |
| 料理メタファー連発 | 原文の例があるならそれを使う |
| 「料理を覚えるとき〜」のような会話的比喩を多用 | 1 セクションに 1 つまでに抑える |
| 「言ってみれば〜のようなものです」 | 削除 or 「〜です」 |

### J. カジュアル過多 / かなじゃくり

| 禁止 | 置換 |
|---|---|
| 「みたいな」「だいたい」「ふんわり」「うっすら」「ぜんぶ」 | 「といった」「およそ」「全部」(漢字) |
| 「やってます」「進んでます」(口語の縮約) | 「やっています」「進んでいます」 |
| 「どっち向き?」 | 「どちら向き」「どんな役割か」 |
| 「〜って何?」(節タイトル) | 文中なら可、節タイトルなら「〜とは」推奨 |
| 「〜しちゃう」「〜してくれちゃう」 | 「〜してしまう」「〜する」 |

### K. 文末・リズム

- 「です。です。です。」の連続 → 体言止め、「だ・である」混在で散らす
- 文体は「です・ます」基調を保つ。「である」は数式説明や客観事実だけに限定
- 同じ動詞で文末が並んだら、1 つは別の動詞・体言止め・受動表現に切り替える
- 同じ文末パターンを 3 連続させない

### L. 数の不整合・数え間違い

「N 個の X」と書いた直後の箇条書きやリストの数が合わない症状 (例: 「2 つのログを見る」と書いて 3 件並べる、「代表的な 3 つ」と書いて 4 つ並べる)。

- 数字を含む文を書いたら、後続のリスト/箇条書き/コマンド数と必ず突き合わせて検算
- 確信が無ければ「代表的なもの」のように数を入れない言い回しに逃げる

### M. 機械翻訳調の残り香 (英→日直訳)

英語ドラフトを日本語にしたときに残るパターン。

| 元の英語 | NG 訳 | OK 訳 |
|---|---|---|
| "It makes sense" | 「センスを作る」(誤訳) | 「理にかなう」 |
| "you can think of X as Y" | 「X を Y と考えてくれます」 | 「X は Y のようなものだ」 |
| "this is important" | 「これは大切です」連発 | 重要性は内容で示し、語句は削除 |
| "Let's dive into" | 「飛び込んでみましょう」 | 削除、または「見ていきます」 |
| "Note that" | 「注意してください、〜です」 | 「ただし、〜」「(注: 〜)」 |
| "In other words" | 「別の言葉で言えば、〜」 | 「つまり、〜」(1 文に 1 つまで) |
| "If you want to ..." | 「もしあなたが〜したいのであれば」 | 「〜したい場合は」 |
| "make sure to ..." | 「〜することを確認してください」 | 「〜してください」 |

### N. 受動・冗長表現の濫用

| 禁止 | 置換 |
|---|---|
| 「〜することが可能です」 | 「〜できます」 |
| 「〜することができます」 (連発時) | 「〜できます」 (一部) or 「〜します」(能動) |
| 「〜が行われます」 | 「〜が起こります」「〜します」 |
| 「〜が必要となります」 | 「〜が必要です」 |
| 「〜を行います」 | 「〜します」 |
| 「実施することにより」 | 「実施すると」「実施して」 |

### O. 一覧化の過多

箇条書きにする必要のない内容まで箇条書きで整列させる症状。AI 出力に頻出。

- 文章で書ける論理的な流れを、無理に箇条書きにしていないか確認
- 1 項目あたり 1 単語〜短句しか書けないなら、それは箇条書きではなく文章にすべき
- 並列のテーマが本当にあるか、見出しと内容を見直す

## 一括チェック用 grep

ファイルを書いた直後、または監査時に必ず回す。

```bash
grep -nE '(まずスタート地点|気持ちで読み解|嬉しいことが[0-9]?|ハードルが高め|イメージです|なイメージ|ですよね|いかがでしたか|ぜひ|驚くほど|展開できます|登場します|と呼ばれるもの|にあたります|に該当します|と考えられます|これにより|やたら|うっすら|ふんわり|ピンとくる|腑に落ちる|ミソ|主役|どっち向き|みたいな比率|だいたいこんな|ぜんぶ|教えてくれます|出してくれる|答えてくれます|することが可能|することができます|を行います|お疲れさまでした|お読みいただき)' <file>
```

ヒットした行はすべて確認し、上記カテゴリの表に従って置換する。

## 運用ルール

1. **draft → grep → fix → review** の順を守る。grep を経ないドラフトはレビューに出さない
2. **置換は意味を保つ**。元の意味が変わりそうな場合は、その場で言い回しを再構成する
3. **過度な漢字化は避ける**。「全て」「事」「物」を漢字に直しすぎると別種の不自然さが出る。接続詞・代名詞・形式名詞はかなを基本に
4. **「です・ます」基調 + 体言止め + 「だ・である」少量** が技術ブログの標準。論文調の「である」連発は避ける
5. **同じ文末パターンを 3 連続させない**
6. **ユーザーの実フィードバックは絶対正** — 辞書に無いパターンを指摘されたら、まず該当箇所を直し、次に辞書に追加する
7. **過剰な改変は禁止**。意味を変えるレベルの書き直しはユーザーに確認する
