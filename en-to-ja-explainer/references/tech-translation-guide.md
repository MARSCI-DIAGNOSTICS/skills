## 略号の表記

- **初出**: 日本語訳 (英語 / 略号) — 例: 行動模倣 (Behavior Cloning, BC)
- **2 回目以降**: 略号のみで可
- 略号の説明はタイトルや見出しでは省略可

## 数式の扱い

- 論文の式番号を引用するときは「論文の式 (2)」のように原文どおり
- 変数記号 (P, ᾱ, τ など) はそのまま使い、初出で読みを併記 (例: τ (タウ))
- 式は別行 + 中央寄せ + 軽い枠で囲って独立させると読みやすい
- 不等号や記号にはスペースを入れる (`x > 0`, `f(x) = y`)

## コードブロックと引用

- 引数のフラグは `--flag-name` の形で `<code>` 化する
- パス・関数名・モジュール名・ファイル名は `<code>` で囲う
- 長いコードブロックは言語名を `<pre><code class="language-xxx">` で指定 (シンタックスハイライト対応)

## 文体方針

### 基本トーン

- **「です・ます」基調**: 説明文の標準
- **体言止め**: 文末リズムの単調さを避ける目的で時々
- **「だ・である」**: 数式説明や客観的事実の記述に限定。連発しない
- **会話的トーン (「面白いのは〜」「実はね、〜」) は禁止**

### 1 文の長さ

- 目安 40-50 字、最大 80 字
- 主語の繰り返しは省略 (「SARM は〜です。SARM は〜します」 → 「SARM は〜で、〜します」)
- 1 文に「で」「に」「は」が 3 つ以上入ったら、文を分ける

### 句読点

- 「、」は 1 文に 2 つまで
- 並列は「、」より「・」を使う (「画像・音声・テキスト」)

### 漢字 / かなのバランス

| かなを基本 | 漢字を基本 |
|---|---|
| すべて、こと、もの、ため (形式名詞) | 設定、確認、構築、評価 (動作名詞) |
| また、さらに、または (接続詞) | |
| とき、ところ、はず | |
| 〜してしまう、〜していく | |

### 否定表現

- 「〜できない」「〜しない」は素直に使う
- 二重否定 (「〜なくはない」) は避ける

## 「やさしく解説」のさじ加減

「初学者でも分かる」を狙う場合でも、子供向け書籍のトーンには寄せない。基準:

- 比喩は 1 セクションに 1 つまで。原文の例 (タオル折り等) を引用するのは OK
- 「面白いのは〜」「実はね、〜」「みなさん」のような会話的トーンは禁止
- 「!」「?」の連発も避ける
- 「と思います」「のではないでしょうか」 → 「です」「になります」に置換

## NG 翻訳パターン

| 元の英語 | NG 訳 | OK 訳 |
|---|---|---|
| "It makes sense" | 「センスを作る」(誤訳) | 「理にかなう」 |
| "you can think of X as Y" | 「X を Y と考えてくれます」 | 「X は Y のようなものだ」 |
| "this is important" | 「これは大切です」連発 | 重要性は内容で示し、語句は削除 |
| "Let's dive into" | 「飛び込んでみましょう」 | 削除、または「見ていきます」 |
| "Note that" | 「注意してください、〜です」 | 「ただし、〜」「(注: 〜)」 |
| "In other words" | 「別の言葉で言えば、〜」 | 「つまり、〜」(1 文に 1 つまで) |

## 用語の一貫性チェック

最終確認で、文書全体に対して:

1. 同じ概念が違うカタカナで書かれていないか (「サブタスク」「サブ・タスク」「subtask」混在)
2. 略号が初出時に展開されているか
3. 数式記号と本文の表記が揃っているか (本文「タウ」、式「τ」のような場合は読みを併記)
