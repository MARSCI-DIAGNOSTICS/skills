# Presentation Preparation Checklist

Comprehensive checklists for preparing and delivering technical presentations at every stage.

## Week Before

### Content Preparation

- [ ] Outline complete with What-Why-How structure
- [ ] All slides drafted
- [ ] One idea per slide verified
- [ ] Code examples simplified and highlighted
- [ ] Diagrams created and labeled
- [ ] Speaker notes added (if using)
- [ ] Transitions between sections smooth
- [ ] Strong hook/opening prepared
- [ ] Clear call-to-action at end
- [ ] 3 key takeaways identified

### Demo Preparation (if applicable)

- [ ] Demo environment created and isolated
- [ ] Demo script written with exact steps
- [ ] Realistic demo data populated
- [ ] All dependencies documented
- [ ] Full demo run-through completed (3x minimum)
- [ ] Backup materials created (video, screenshots)
- [ ] Recovery scripts tested

### Logistics

- [ ] Venue/platform confirmed
- [ ] Equipment requirements known
- [ ] Time slot confirmed
- [ ] Presentation format confirmed (slides, aspect ratio)
- [ ] Recording requirements known
- [ ] Q&A format understood

### Feedback Round

- [ ] Dry run with colleague completed
- [ ] Feedback incorporated
- [ ] Timing adjusted based on practice
- [ ] Confusing sections clarified

## Day Before

### Content Finalization

- [ ] Final read-through of all slides
- [ ] Typos and grammar checked
- [ ] Font sizes verified (24pt+ body, 18pt+ code)
- [ ] All links and resources working
- [ ] Speaker notes finalized
- [ ] Timing confirmed (with buffer)

### Technical Setup

- [ ] Slides exported to backup format (PDF)
- [ ] Slides uploaded to cloud backup (Google Drive, Dropbox)
- [ ] Slides on USB drive (physical backup)
- [ ] Slides emailed to self
- [ ] Demo environment verified working
- [ ] Backup demo materials accessible
- [ ] Laptop fully charged
- [ ] Presentation mode tested

### Personal Preparation

- [ ] Full practice run completed
- [ ] Outfit selected and ready
- [ ] Travel route planned (if in-person)
- [ ] Water bottle ready
- [ ] Get good sleep

## Morning Of

### Technical Verification

- [ ] All devices charged (laptop, phone, backup)
- [ ] Slides open and working
- [ ] Demo environment tested
- [ ] Backup materials accessible
- [ ] USB drive in bag
- [ ] Presentation clicker working (batteries fresh)

### Notification Management

**Laptop:**

- [ ] Focus mode/Do Not Disturb enabled
- [ ] Email client closed
- [ ] Slack/Teams set to DND
- [ ] All chat apps closed
- [ ] Calendar notifications silenced
- [ ] Browser notifications disabled
- [ ] System updates postponed

**Phone:**

- [ ] Silent mode enabled
- [ ] Notifications fully disabled
- [ ] Phone face-down or in bag during presentation

### Screen Preparation

- [ ] Desktop cleaned of personal items
- [ ] Irrelevant browser tabs closed
- [ ] Browser bookmarks bar hidden (if personal)
- [ ] Zoom/font size increased for visibility
- [ ] Screen resolution appropriate for venue
- [ ] Terminal/IDE font size increased (18pt+)

## Right Before (30 Minutes)

### Venue/Platform Check (In-Person)

- [ ] Arrive 30+ minutes early
- [ ] Test laptop connection to projector
- [ ] Verify display mirroring/extended desktop
- [ ] Check audio if using video
- [ ] Locate power outlet
- [ ] Test presentation clicker
- [ ] Check lighting for visibility
- [ ] Locate water/refreshments
- [ ] Identify emergency exits (calm nerves)
- [ ] Locate bathroom

### Venue/Platform Check (Remote)

- [ ] Test video platform (Zoom, Teams, etc.)
- [ ] Verify screen sharing works
- [ ] Test audio levels
- [ ] Check camera framing
- [ ] Confirm backup connection method
- [ ] Close unnecessary applications
- [ ] Verify network connection stable
- [ ] Position water within reach (off-camera)

### Mental Preparation

- [ ] Review opening hook (know it cold)
- [ ] Review key transitions
- [ ] Review closing and call-to-action
- [ ] Deep breaths
- [ ] Quick stretch
- [ ] Positive visualization

## During Presentation

### Opening (First 2 Minutes)

- [ ] Start with hook (not "Thank you for having me")
- [ ] Establish relevance for audience
- [ ] Set expectations for what they'll learn
- [ ] Project confidence (even if nervous)

### Body (Main Content)

- [ ] Maintain eye contact with audience
- [ ] Vary pace and energy
- [ ] Use planned pauses
- [ ] Check timing at landmarks
- [ ] Adjust if running long (skip optional content)
- [ ] Watch audience for confusion
- [ ] Drink water as needed

### Demo (If Applicable)

- [ ] Narrate continuously
- [ ] Type/click deliberately (slow down)
- [ ] Follow script
- [ ] Use 30-second rule for issues
- [ ] Switch to backup if needed
- [ ] Continue with confidence

### Q&A

- [ ] Repeat questions for audience
- [ ] Pause before answering
- [ ] Acknowledge good questions
- [ ] "I don't know, I'll find out" is acceptable
- [ ] Defer off-topic to offline
- [ ] End on time

## After Presentation

### Immediate

- [ ] Thank the organizers
- [ ] Be available for follow-up questions
- [ ] Collect any promised follow-ups
- [ ] Exchange contact info as appropriate

### Same Day

- [ ] Note what worked and what didn't
- [ ] Record any follow-up commitments
- [ ] Download/save recording if available
- [ ] Save chat/questions for future reference

### Within Week

- [ ] Send slides to organizers (if promised)
- [ ] Complete any follow-up commitments
- [ ] Update slides based on feedback
- [ ] Write brief retrospective notes
- [ ] Add experience to speaking portfolio
- [ ] Request recording if promised

## Platform-Specific Checklists

### In-Person Conference Talk

Additional considerations:

- [ ] Arrive day before if traveling
- [ ] Test in actual room if possible
- [ ] Meet A/V staff
- [ ] Understand transition between speakers
- [ ] Know how to adjust mic
- [ ] Understand recording setup
- [ ] Have business cards ready
- [ ] Plan for post-talk networking

### Internal Team Presentation

Additional considerations:

- [ ] Slides accessible on company platform
- [ ] Calendar invite with correct link
- [ ] Pre-read materials distributed (if any)
- [ ] Know who's dialing in vs in-person
- [ ] Hybrid setup tested (camera, screen share)
- [ ] Recording consent confirmed
- [ ] Follow-up meeting scheduled (if needed)

### Virtual Conference/Webinar

Additional considerations:

- [ ] Backup internet connection ready (mobile hotspot)
- [ ] Quiet environment secured
- [ ] Professional background (physical or virtual)
- [ ] Good lighting on face
- [ ] Camera at eye level
- [ ] Wired connection preferred
- [ ] Test with specific platform (Zoom, Hopin, etc.)
- [ ] Understand green room/backstage process
- [ ] Know how audience will ask questions

### Lightning Talk (5-10 Minutes)

Additional considerations:

- [ ] Ruthlessly cut content (time is tiny)
- [ ] No demo OR pre-recorded only
- [ ] Practice with timer (multiple times)
- [ ] Know exactly how you'll be cut off
- [ ] Single, focused message only
- [ ] Slides advance quickly (many short slides)

## Emergency Procedures

### Slides Won't Load

1. Try backup USB drive
2. Try cloud backup (from any device)
3. Use PDF export
4. Present from phone/tablet in emergency
5. Present without slides (you know the content)

### Projector/Display Issues

1. Try different cable/adapter
2. Try different laptop (load from cloud)
3. Restart projector
4. Adjust to projector's native resolution
5. Use TV/monitor if available

### Demo Fails

1. Try one quick fix (30 seconds)
2. Switch to backup video
3. Use screenshot walkthrough
4. Describe verbally and move on
5. Offer follow-up after the session

### Lost Voice/Sick

1. Have water handy
2. Use microphone if available
3. Slow down and speak deliberately
4. If severe, honestly address it
5. Have backup presenter identified if possible

### Running Over Time

1. Skip optional sections (identified in advance)
2. Summarize remaining points quickly
3. Offer follow-up for skipped content
4. Always finish on time (respect audience)

### Tech Failure (Complete)

1. Stay calm
2. Acknowledge the situation
3. Pivot to conversation/Q&A
4. Offer to reschedule or provide materials
5. Maintain professionalism throughout

## Speaking Energy Checklist

Before you go on:

- [ ] Shoulders back, posture open
- [ ] Take 3 deep breaths
- [ ] Smile (even if nervous)
- [ ] Remember: audience wants you to succeed
- [ ] Focus on helping the audience
- [ ] You know this content (you prepared)
- [ ] Mistakes are okay and recoverable
- [ ] You've got this
